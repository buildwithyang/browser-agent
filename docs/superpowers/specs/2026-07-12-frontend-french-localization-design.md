# Frontend 法语本地化设计

日期：2026-07-12

## 背景

Agent Bridge 当前前端支持中文和英文。核心用户仍以使用英语求职的中国留学生为主，但法国已有种子用户，因此先增加法语前端界面有明确验证价值。

阿拉伯语暂不加入：UAE 求职场景可继续由英语覆盖，且阿拉伯语需要额外承担 RTL 布局和完整视觉回归成本。目前没有足够的真实用户需求支撑这项投入。

## 目标

- `frontend/` 的公开落地页和登录后简历管理界面完整支持法语。
- 浏览器首选语言为法语时，首次访问自动显示法语。
- 用户可以通过清晰的语言下拉框在中文、English、Français 之间切换。
- 用户选择持久化到浏览器本地存储。
- 法语界面的日期按法国格式显示。
- 保持当前深色工业化视觉，不因为增加语言产生明显布局退化。

## 非目标

- 不修改 Chrome 扩展的语言选项或界面文案。
- 不修改 Gateway、任务 schema 或 AI 输出语言。
- 不增加阿拉伯语或 RTL 支持。
- 不翻译 `frontend/public/privacy.html`。隐私政策属于法律文本，后续应经过单独的准确性审阅。
- 不引入路由级语言 URL，例如 `/fr`。

## 交互设计

现有中英文循环按钮替换为紧凑的原生语言下拉框。选项固定为：

- 中文
- English
- Français

下拉框继续使用当前 `.btn-ghost` / `.lang-toggle` 的视觉语言：深色透明背景、细边框、等宽小字号和暖黄色聚焦状态。原生 `select` 保留键盘操作、屏幕阅读器和移动端选择器能力。

语言选择器在落地页导航和登录后顶栏使用同一组件。控件具备可翻译的无障碍标签，不依赖当前选项本身表达用途。

## 国际化结构

### 支持语言

`SUPPORTED` 扩展为 `zh`、`en`、`fr`。语言元数据集中维护显示名称和日期 locale，避免组件内继续写二元判断。

### 首选语言检测

优先级保持为：

1. `localStorage.ab_lang` 中有效的已保存值。
2. `navigator.languages` 中第一个可以映射到支持语言的值。
3. `navigator.language`。
4. 英文兜底。

`zh-*` 映射到 `zh`，`fr-*` 映射到 `fr`，`en-*` 映射到 `en`。其他语言回退英文，因为产品主要面向国际求职用户。

### HTML 元数据

语言变化时同步设置 `document.documentElement.lang`：

- `zh` → `zh-CN`
- `en` → `en`
- `fr` → `fr`

本期方向始终为 `ltr`，无需设置 RTL。

### 文案回退

法语文案缺失时先回退英文，再回退中文，最后显示 key。中文和英文现有行为保持兼容。

所有法语用户可见文案必须在本期补齐，回退只用于防止未来新增 key 导致界面直接显示内部标识，不作为交付时遗漏翻译的手段。

## 法语覆盖范围

`frontend/src/strings.js` 新增完整 `fr` 字典，覆盖：

- 顶部导航、登录、退出和 Chrome 安装入口。
- 落地页主标题、副标题、步骤、能力卡片、CTA 和匹配仪表盘。
- 简历上传、解析状态、错误、成功提示和简历列表操作。
- 浏览器扩展连接卡片的状态、操作和诊断信息。
- 隐私政策入口名称，但不翻译隐私政策正文。

翻译使用法国法语，统一使用 `CV`，避免在求职语境中使用更偏学术或不自然的表达。Chrome、LinkedIn、Indeed、Agent Bridge 等产品名不翻译。

## 日期与动态值

日期格式由语言元数据映射：

- 中文：`zh-CN`
- 英文：`en-US`
- 法语：`fr-FR`

继续使用 `hour12: false`。文件大小保持现有单位格式；字符数通过法语字典中的占位符文案显示。

## 测试

新增国际化单元测试，至少覆盖：

- 已保存语言优先于浏览器语言。
- `fr-FR` 和 `fr-CA` 均检测为法语。
- 不支持的浏览器语言回退英文。
- 设置法语会持久化并更新 HTML `lang`。
- 法语字典与英文基准拥有相同的叶子 key 和数组结构。
- 法语缺失 key 时按英文、中文顺序回退。
- 日期 locale 映射包含 `fr-FR`。

运行现有前端测试和生产构建，确认国际化改动没有破坏扩展连接逻辑或 Vite 构建。

## 文档

更新 `frontend/README.md`，说明前端支持中文、英文和法语，记录自动检测、手动选择和本地持久化行为，并明确扩展与 AI 输出语言不在本期范围。

## 验收标准

1. 法语浏览器首次进入前端时自动显示法语。
2. 用户可从下拉框选择中文、English 或 Français，刷新后选择保持。
3. 落地页、简历管理页和扩展连接卡片没有缺失的法语文案。
4. 法语日期使用 `fr-FR` 格式。
5. 桌面和移动宽度下，语言选择器不会挤压主要 CTA 或用户操作。
6. `npm test` 与 `npm run build` 通过。
